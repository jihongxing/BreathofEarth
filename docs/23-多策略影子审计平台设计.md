# 多策略影子审计平台设计

> 本文件记录多策略影子观察平台的目标架构。它不是新增实盘策略授权，也不是杠杆批准。
>
> 当前项目判定仍是：`Research PASS / Production design APPROVED / Live leveraged execution NOT YET APPROVED`。
>
> 实现状态：第一版已经端到端落地。策略注册表、分层审计结果模型、多策略 shadow runner、只读 API 与前端对比面板均已接入。该状态不改变任何实盘、杠杆或下单边界。

## 一、目标

Stage 9.5 原本只观察当前 90/10 生产候选。当前第一版已经升级为多策略影子审计平台：

- 同时观察多套策略。
- 使用同一套干净数据校验。
- 使用同一套交易摩擦、税务、券商利差、失败折价模型。
- 使用同一套影子账本与准入门禁。
- 横向比较谁在真实世界扣完成本后仍值得进入人工评审。

核心问题不是“哪条曲线最好看”，而是：

> 哪套策略在干净数据、真实摩擦、税务压力、运行失败折价和只读影子观察之后，仍然满足可治理边界？

## 二、非目标

本平台不做以下事情：

- 不产生真实订单。
- 不连接自动下单执行器。
- 不把 `READY_FOR_HUMAN_REVIEW` 解释成实盘批准。
- 不允许前端出现一键交易、一键加杠杆、一键 Shadow 转 Live。
- 不为了提高 CAGR 而放松生产候选的 10% 卫星上限、MA150、recovery、滑点或 fail-closed 数据校验。
- 不把税务模型当成税务意见。
- 不把券商利差模型当成 IBKR 或其他券商真实对账单。

## 三、分层账本

每套策略必须输出同一组层级指标：

| 层级 | 名称 | 含义 |
|------|------|------|
| L0 | `data_validated` | 数据通过非正价格、缺失、陈旧和资产对齐校验 |
| L1 | `research_gross` | 干净数据研究净值，含策略自身再平衡逻辑 |
| L2 | `execution_adjusted` | 加入交易成本、滑点、冲击成本 |
| L3 | `broker_adjusted` | 加入券商现金/融资利差、借贷成本或现金收益折价 |
| L4 | `tax_adjusted` | 加入股息预扣税、分红税、实体情景税率 |
| L5 | `failure_adjusted` | 加入漏执行、延迟执行、数据中断、券商不可用、运行失败折价 |
| L6 | `shadow_observed` | 接入 Stage 9.5 只读影子观察与保证金快照 |
| L7 | `admission_gated` | 输出 `NOT_APPROVED / READY_FOR_HUMAN_REVIEW` |

任何策略如果某一层缺失，后续层不得推导为安全，只能标记为 `UNAVAILABLE / PARTIAL / NOT_APPROVED`。

## 四、策略注册模型

已实现：

```text
engine/strategy_registry.py
tests/test_strategy_registry.py
```

每个策略注册项使用统一 schema：

```python
{
    "strategy_id": "production_90_10",
    "display_name": "90% Defensive Core + 10% Satellite",
    "status": "production_candidate",
    "confidence": "FULL_CYCLE_AUDITED",
    "assets": ["SPY", "TLT", "GLD", "SHV", "QQQ"],
    "base_currency": "USD",
    "data_policy": "yahoo_adj_close_clean",
    "risk_policy": {
        "max_research_mdd": -0.15,
        "max_failure_adjusted_mdd": -0.16,
        "satellite_cap": 0.10,
    },
    "live_execution_allowed": False,
}
```

首批策略建议只纳入四类：

| strategy_id | 定位 | 说明 |
|-------------|------|------|
| `classic_permanent_portfolio` | 研究基线 | 经典 25/25/25/25 永续投资组合 |
| `fixed_defensive_core` | 防守锚 | MA150 + 8w + 固定防御核心 |
| `production_90_10` | 当前主线 | 当前生产候选，默认唯一可进入 Stage 9.5 主观察 |
| `benchmark_balanced_proxy` | 外部参照 | 如 VBIAX/PRPFX/RPAR 可用窗口的对照 |

现金替代、趋势卫星、CTA、SMH 等实验不得直接进入生产候选，只能先作为 `research_candidate`。

## 五、审计流水线

已实现：

```text
engine/strategy_audit_pipeline.py
tests/test_strategy_audit_pipeline.py
```

职责：

1. 从 registry 读取策略定义。
2. 加载冻结或实时数据输入。
3. 执行数据 fail-closed 校验。
4. 生成 L1 研究净值。
5. 逐层应用摩擦模型。
6. 输出统一 `StrategyAuditResult`。

推荐输出：

```python
{
    "strategy_id": "production_90_10",
    "as_of": "2026-06-23",
    "status": "PASS",
    "layers": {
        "research_gross": {
            "cagr": 0.0760,
            "mdd": -0.1395,
            "final_nav": 476436.35,
        },
        "tax_adjusted": {
            "cagr": 0.0703,
            "mdd": -0.1456,
            "final_nav": 426022.12,
        },
        "failure_adjusted": {
            "cagr": 0.0660,
            "mdd": -0.1510,
            "final_nav": 390489.22,
        },
    },
    "blockers": [],
    "warnings": [
        "calibrated friction matrix is not a broker statement",
        "tax model is not tax advice",
    ],
}
```

## 六、摩擦模型

建议新增：

```text
engine/friction_model.py
```

摩擦模型必须参数化，不允许散落在策略代码里。

基础组件：

- `ExecutionCostModel`
  - 常规换手成本
  - 恐慌滑点
  - ETF spread
  - 盘口冲击模型
- `BrokerCarryModel`
  - 现金收益折价
  - 保证金融资成本
  - 券商现金/融资负利差
- `TaxWithholdingModel`
  - SPY/QQQ 等分红预扣税
  - SHV/债券 ETF 分配口径
  - 实体税率情景
- `OperationalFailureModel`
  - 漏执行折价
  - 延迟执行折价
  - 数据不可用折价
  - 券商不可用折价

模型输出必须是逐日 dollar ledger：

```text
start_nav + strategy_pnl - execution_cost - broker_cost - tax_cost - failure_cost = end_nav
```

不允许只用一个全局年化 haircut 充当真实摩擦。

## 七、多策略影子观察

已实现：

```text
live/multi_strategy_shadow_runner.py
tests/test_multi_strategy_shadow_runner.py
```

职责：

- 对每个策略生成目标权重。
- 计算目标名义本金。
- 计算相对当前持仓的影子换手。
- 读取只读报价与盘口深度。
- 读取保证金字段。
- 写出多策略影子报告。

输出文件：

```text
data/shadow/multi_strategy_shadow_YYYYMMDD_HHMMSS.json
data/shadow/latest_multi_strategy_shadow.json
```

示例结构：

```python
{
    "timestamp": "2026-06-23T00:00:00Z",
    "trading_disabled": True,
    "live_leverage_approved": False,
    "strategies": {
        "production_90_10": {
            "status": "ATTENTION",
            "target_weights": {},
            "shadow_turnover": {},
            "slippage_audit": {},
            "margin_snapshot": {},
            "admission_status": "NOT_APPROVED",
        }
    }
}
```

`live/` 目录仍然只能观察，不能调用 `place_order`、`cancel_order` 或任何交易确认流程。

## 九、准入门禁

每套策略独立输出 admission：

```python
{
    "strategy_id": "production_90_10",
    "status": "NOT_APPROVED",
    "level": "warning",
    "live_leverage_approved": False,
    "human_review_required": True,
    "blockers": [
        {
            "code": "missing_tax_adjusted_layer",
            "message": "tax-adjusted ledger is unavailable"
        }
    ]
}
```

只有同时满足以下条件，才允许输出 `READY_FOR_HUMAN_REVIEW`：

- 数据层完整且 fail-closed 校验通过。
- 分层账本至少包含 `research_gross / execution_adjusted / broker_adjusted / tax_adjusted / failure_adjusted`。
- MDD 未突破该策略自己的风险红线。
- Stage 9.5 观察窗口满足要求。
- 券商只读快照和保证金字段覆盖率达标。
- `live_leverage_approved` 仍为 `false`。

`READY_FOR_HUMAN_REVIEW` 只代表可以进入人工评审，不代表实盘、杠杆、下单批准。

- `python -m pytest -q` 全量通过。
