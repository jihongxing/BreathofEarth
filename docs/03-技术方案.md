# 技术方案

## 设计原则

- 单体优先，拒绝过早微服务化
- 用最简单的技术栈解决问题
- 鲁棒性和安全性是第一准则
- 性能不是瓶颈（小规模使用，日频操作）

## 技术栈

| 层面 | 选择 | 理由 |
|------|------|------|
| 语言 | Python 3.12+ | 生态最丰富，金融库最多，开发最快 |
| 数据库 | SQLite（开发/自用）→ PostgreSQL（多人） | 零运维，单文件，够用 |
| Web框架 | FastAPI | 轻量，异步，自带文档 |
| 定时任务 | APScheduler | 内嵌调度，不需要外部依赖 |
| 数据分析 | pandas + numpy | 回测、指标计算 |
| 可视化 | matplotlib / plotly | 回测结果展示 |
| 前端（可选） | 简单 HTML + Chart.js 或 Streamlit | 看板，不需要复杂前端 |

## 数据模型（精简版）

```sql
-- 账户
CREATE TABLE accounts (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    base_currency TEXT DEFAULT 'USD',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 组合
CREATE TABLE portfolios (
    id TEXT PRIMARY KEY,
    account_id TEXT REFERENCES accounts(id),
    state TEXT DEFAULT 'IDLE',
    target_core REAL DEFAULT 0.75,
    target_stability REAL DEFAULT 0.15,
    target_alpha REAL DEFAULT 0.10,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 资产定义
CREATE TABLE assets (
    id TEXT PRIMARY KEY,
    symbol TEXT UNIQUE NOT NULL,
    name TEXT,
    asset_class TEXT  -- EQUITY / BOND / GOLD / CASH
);

-- 持仓
CREATE TABLE positions (
    id TEXT PRIMARY KEY,
    portfolio_id TEXT REFERENCES portfolios(id),
    layer TEXT NOT NULL,  -- CORE / STABILITY / ALPHA
    asset_id TEXT REFERENCES assets(id),
    quantity REAL DEFAULT 0,
    avg_price REAL DEFAULT 0,
    target_weight REAL DEFAULT 0,
    current_weight REAL DEFAULT 0
);

-- 交易记录
CREATE TABLE transactions (
    id TEXT PRIMARY KEY,
    portfolio_id TEXT REFERENCES portfolios(id),
    type TEXT NOT NULL,  -- DEPOSIT / WITHDRAWAL / BUY / SELL / REBALANCE
    asset_id TEXT,
    amount REAL,
    quantity REAL,
    price REAL,
    status TEXT DEFAULT 'PENDING',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 风控事件
CREATE TABLE risk_events (
    id TEXT PRIMARY KEY,
    portfolio_id TEXT REFERENCES portfolios(id),
    type TEXT NOT NULL,  -- CORRELATION_BREAK / DRAWDOWN / LIQUIDITY / ALPHA_STOP
    severity TEXT,
    action_taken TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 状态快照（用于回放和审计）
CREATE TABLE state_snapshots (
    id TEXT PRIMARY KEY,
    portfolio_id TEXT REFERENCES portfolios(id),
    date TEXT NOT NULL,
    portfolio_state TEXT,
    nav REAL,
    core_weight REAL,
    stability_weight REAL,
    alpha_weight REAL,
    drawdown REAL,
    snapshot_data TEXT,  -- JSON，完整快照
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## API 接口（极简）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/deposit | 入金 |
| POST | /api/withdraw | 出金 |
| GET  | /api/portfolio/{id} | 查看组合状态 |
| GET  | /api/portfolio/{id}/history | 历史快照 |
| GET  | /api/risk/{portfolio_id} | 风控状态 |
| POST | /api/backtest/run | 运行回测（开发阶段） |

## 部署

自用阶段：本地运行，`python main.py` 即可。

如需远程访问：一台最小规格云服务器 + Docker 单容器部署。

不需要 Kubernetes、不需要消息队列、不需要微服务。
