# CAGR 提升研究计划

> 本文件定义 `codex/cagr-uplift-with-mdd-guardrails` 分支的研究边界。
> 它不是生产策略变更，不是实盘授权，也不是提高卫星仓位或放松风控阈值的许可。

## 一、研究目标

当前 90/10 生产候选在第一版真实摩擦压力审计下：

| 口径 | CAGR | MDD | 定位 |
|------|------|-----|------|
| `research_current` | `7.60%` | `-13.95%` | 研究净值口径 |
| `unlevered_base_case` | `5.75%` | `-15.32%` | 第一真实摩擦参考口径 |
| `unlevered_harsh_case` | `3.53%` | `-16.99%` | 保守压力口径 |

本阶段目标不是继续美化 `research_current`，而是寻找能把 `unlevered_base_case` 的 CAGR 推向 `7% - 8%` 的新增收益来源。

硬约束：

- 不得牺牲最大回撤边界。
- 新方案的 `unlevered_base_case` MDD 不得劣于当前 `-15.32%`。
- 优先目标是把 MDD 维持在 `-15%` 附近，同时提升净后参考收益。
- 任何超过 `-16%` MDD 的方案，默认失败，除非后续治理文档单独批准更高承伤边界。

## 二、不可触碰的红线

以下路线本阶段不允许作为收益提升手段：

- 放松 `MA150`、急性危机、慢性宏观防御或 fail-closed 数据校验。
- 把 10% 卫星仓位提高到 15%、20% 或更高。
- 把 `SMH` 直接加入生产候选。
- 用滚动择参、窗口搜索或复杂条件组合追求漂亮曲线。
- 把 `7.60% / -13.95%` 写成税后、券商后、故障后的真实收益。
- 把 Stage 9.5 从只读观察层改造成交易执行层。
- 让前端出现一键加杠杆、一键交易或 Shadow 转 Live 的入口。

已审计且暂不重复投入的方向：

- 单纯替换 `TLT` 为 `IEF / AGG`。
- 宏观防御期定向切除 `TLT` 并保留 `SPY / GLD`。
- 阶梯恢复期平滑。
- 继续压低波动率或相关性触发阈值。

这些方向的边际收益不足，且更容易把系统重新拖回过拟合。

## 三、优先实验矩阵

新增收益来源必须满足三个特征：

- 可解释：收益来自清晰资产或现金流机制，而不是参数碰撞。
- 低相关：不能在 2008、2020、2022 与主组合同步塌方。
- 低治理成本：不能显著增加换手、税务复杂度、券商执行复杂度或前端交易冲动。

### 1. 现金替代与票息增强

研究问题：

- 当前 `SHV` 和现金口径是否低估了真实可获得的短端收益？
- 能否用更合适的短端国债或货币基金代理改善现金 sleeve 的净后收益？

候选标的：

- `SGOV`
- `BIL`
- `USFR`
- `TFLO`

通过标准：

- `unlevered_base_case` CAGR 明显高于 `5.75%`。
- MDD 不劣于 `-15.32%`。
- 不引入高久期风险。
- 真实摩擦文档必须说明股息、利息、预扣税和券商现金利息之间的口径差异。

第一轮旁路审计结果：

命令：

```bash
python -m backtest.cagr_uplift_audit
```

脚本：

- `backtest/cagr_uplift_audit.py`
- `tests/test_cagr_uplift_audit.py`

冻结输入：

- `data/audit_snapshots/2026-06-23-cash-proxy/`

同窗口、同本金、同 90/10 拓扑、同真实摩擦基础口径下：

| 现金代理 | 窗口 | Real CAGR | Real MDD | 同窗口 SHV CAGR | 同窗口 SHV MDD | CAGR 增量 | MDD 增量 |
|----------|------|-----------|----------|-----------------|----------------|-----------|----------|
| `BIL` | `2007-05-30..2026-04-30` | `5.87%` | `-15.61%` | `5.92%` | `-15.73%` | `-0.04%` | `+0.12%` |
| `SGOV` | `2020-06-01..2026-04-30` | `10.84%` | `-17.57%` | `10.79%` | `-17.76%` | `+0.05%` | `+0.18%` |
| `USFR` | `2014-02-04..2026-04-30` | `7.30%` | `-16.56%` | `7.28%` | `-16.86%` | `+0.03%` | `+0.30%` |
| `TFLO` | `2014-02-04..2026-04-30` | `7.28%` | `-16.58%` | `7.28%` | `-16.86%` | `+0.01%` | `+0.28%` |

结论：

- `SGOV / USFR / TFLO` 的方向是对的，MDD 同窗口略有改善。
- 但 CAGR 增量只有 `1bp - 5bp` 量级，不足以把 `unlevered_base_case` 从 `5.75%` 推向 `7%+`。
- `BIL` 比 `SHV` 更像纯现金，但同窗口 CAGR 反而略低。
- 现金替代可以作为微调治理记录，不能作为本阶段主要破局点。
- 短历史代理不能被包装成 2005 起的全周期结论。

### 2. 小比例趋势跟随 Sleeve

研究问题：

- 是否可以在不提高总卫星仓位的前提下，把 10% 卫星内部的一部分改成低相关趋势跟随？
- 趋势 sleeve 是否能在 2008/2022 提供正凸性，而不是只在牛市增加 beta？

候选设计：

- 保持总卫星 `10%` 不变。
- 在卫星内部测试 `QQQ / SPY / GLD` 与趋势资产的替换比例。
- 趋势规则必须固定、少参数、低换手。

可测试方向：

- `DBMF` 或类似管理期货 ETF，只允许作为旁路审计输入。
- 简单 12 月动量的风险资产/现金切换，但必须证明换手和税务压力可控。

通过标准：

- `unlevered_base_case` CAGR 进入 `7%+`。
- MDD 不劣于 `-15.32%`。
- 危机年份切片不能只靠某一年单点贡献。
- Benchmark 对比必须同时展示 `VBIAX / PRPFX / RPAR`。

第二轮旁路审计结果：

命令：

```bash
python -m backtest.cagr_uplift_audit
```

脚本：

- `backtest/cagr_uplift_audit.py`
- `tests/test_cagr_uplift_audit.py`

冻结输入：

- `data/audit_snapshots/2026-06-23-trend-satellite/`

同窗口、同本金、总卫星仍为 `10%`、同真实摩擦基础口径下：

| 趋势卫星方案 | 置信等级 | 窗口 | Real CAGR | Real MDD | 同窗口 Baseline CAGR | 同窗口 Baseline MDD | CAGR 增量 | MDD 增量 | 2022 Return | 2022 MDD |
|--------------|----------|------|-----------|----------|----------------------|---------------------|-----------|----------|-------------|----------|
| `qqq_gld_dbmf` | `LOW_SHORT_HISTORY` | `2019-05-08..2026-04-30` | `8.01%` | `-12.11%` | `8.21%` | `-14.16%` | `-0.21%` | `+2.05%` | `-7.26%` | `-10.77%` |
| `gld_qqq_cash_12m_momentum` | `FULL_WINDOW_RULE_BASED` | `2005-01-03..2026-04-30` | `5.58%` | `-14.71%` | `5.75%` | `-15.32%` | `-0.17%` | `+0.61%` | `-9.16%` | `-13.37%` |

结论：

- 趋势 sleeve 有明确的回撤治理价值，尤其 `DBMF` 短窗口把 MDD 从 `-14.16%` 改善到 `-12.11%`。
- 但两组趋势实验都没有提升真实摩擦 CAGR，反而分别降低 `0.21%` 和 `0.17%`。
- `DBMF` 只能作为短历史、低置信研究记录，不能用 2019 后窗口冒充 2005 全周期。
- `12M QQQ/现金动量` 通过全周期规则审计，但收益下降，说明简单时序动量更像防守刹车，不是收益引擎。
- 第二阶段没有拿到 `7%+` CAGR 突破口，下一步应转向真实摩擦模型校准或寻找更长历史 CTA proxy。

### 3. 税务与券商利差校准

研究问题：

- 当前参数化拖累是否过于保守或过于粗糙？
- 用真实 IBKR 现金利息、融资利率、预扣税和交易报告校准后，净后 CAGR 的可信区间是多少？

候选工作：

- 将股息预扣税、现金利息、融资利率、运行失败折价从静态参数升级为年度序列或场景表。
- 输出 `base / conservative / severe` 三档真实摩擦情景。

通过标准：

- 不能因为校准变乐观就提高生产授权等级。
- 需要保留参数解释和敏感性表。
- Stage 9.5 未完成 60 个交易日之前，所有结果仍是研究口径。

### 4. 低相关收益资产审计

研究问题：

- 是否存在可以小比例加入卫星、但不在危机中显著放大回撤的资产类别？

候选方向：

- 全球低波股票或质量因子 ETF。
- 短久期通胀保护或浮息工具。
- 黄金相关替代只允许先旁路测试，不能直接改 35% 宏观防御承重墙。

通过标准：

- 必须跑全样本、危机切片、真实摩擦压力和 benchmark 对比。
- 不得用上市时间很短的 ETF 直接替代长历史结论。
- 若历史窗口短，文档必须降低结论等级。

## 四、统一审计输出

每个收益增强实验至少输出：

- `research_current`
- `unlevered_base_case`
- `unlevered_harsh_case`
- 2008 / 2020 / 2022 危机切片
- 与 `VBIAX / PRPFX / RPAR` 的双口径对比
- 换手、成本、现金占比、最大单日权重变化
- 数据来源、日期范围、缺失处理和 fail-closed 结果

建议脚本命名：

```bash
backtest/cagr_uplift_audit.py
tests/test_cagr_uplift_audit.py
```

若实验使用新增行情数据，默认不提交 `data/raw/*.csv`。可复现输入必须冻结到：

```text
data/audit_snapshots/<date>-<source>/
```

并附带 `manifest.json`，记录 ticker、日期范围、行数、SHA256 和来源。

## 五、通过与失败标准

方案可以进入下一轮生产候选评审，必须同时满足：

- `unlevered_base_case` CAGR >= `7.00%`
- `unlevered_base_case` MDD >= `-15.32%`
- `unlevered_harsh_case` MDD >= `-17.00%`
- 危机切片没有单点穿透当前候选的治理边界
- 真实摩擦参数清晰可解释
- 前端和 live 脚本仍然只读、fail closed
- 文档明确写出仍不构成实盘杠杆授权

方案应立即失败或降级为研究记录，如果出现：

- 收益提升主要来自 2020 或 2023-2026 单一窗口。
- 换手显著增加但没有实盘执行解释。
- 需要提高卫星总仓位超过 `10%`。
- MDD 恶化但用 CAGR 解释为“值得”。
- 数据窗口短却被当成长周期结论。

## 六、推荐执行顺序

1. 先做现金替代与票息增强审计。
2. 再做小比例趋势跟随 sleeve 审计。
3. 然后把真实摩擦模型从静态参数升级为年度序列。
4. 最后才讨论是否形成新的 90/10 候选变体。

当前最合理的第一步：

```bash
python -m backtest.cagr_uplift_audit
```

但在脚本存在之前，应先实现 `backtest/cagr_uplift_audit.py`，只测试现金替代与票息增强，不改生产引擎。
