# 从仿真到实盘：四步路线图

## 架构概览

```
┌─────────────────────────────────────────────────┐
│              息壤核心引擎                         │
│  Portfolio Engine + Risk Engine + Rebalance      │
│                    │                             │
│                    ▼                             │
│           ┌───────────────┐                      │
│           │ BaseExecutor  │ ← 统一接口            │
│           └───────┬───────┘                      │
│     ┌─────────────┼─────────────┐                │
│     ▼             ▼             ▼                │
│ ┌────────┐  ┌──────────┐  ┌──────────┐          │
│ │ Paper  │  │ Manual   │  │ Broker   │          │
│ │ (仿真) │  │ (人工)   │  │ (券商)   │          │
│ └────────┘  └──────────┘  └────┬─────┘          │
│   Phase 1     Phase 2     Phase 3/4              │
│                                │                 │
│                    ┌───────────┼──────────┐      │
│                    ▼           ▼          ▼      │
│                 ┌──────┐  ┌──────┐  ┌──────┐    │

└─────────────────────────────────────────────────┘
```

核心引擎的逻辑完全不变，只是"执行层"在不同阶段切换不同的实现。

## Phase 1: Paper Trading（当前阶段）

执行器：`PaperExecutor`
配置：`XIRANG_EXECUTOR=paper`（默认）

行为：
- 所有交易都是虚拟的，数据库里改数字
- 用 Yahoo Finance 的收盘价作为"成交价"
- 扣除 0.1% 的模拟摩擦成本

验证目标：
- 状态机逻辑是否正确
- 风控触发是否合理
- 通知推送是否正常
- 报告数据是否准确

周期：3-6 个月

退出条件：
- 运行成功率 > 99%
- 风控触发合理（无误触发）
- 报告指标与回测一致

## Phase 2: 人工执行期

执行器：`ManualExecutor`
配置：`XIRANG_EXECUTOR=manual`

行为：
- 系统生成具体指令："卖出 12 股 SPY，买入 8 股 TLT"
- 通过 Telegram / 微信推送给你
- 你在券商 APP 上手动执行
- 执行完成后在系统中确认，录入实际成交价

新增能力：
- 具体股数计算（目标金额 / 当前价格，取整）
- 先卖后买排序（卖出回笼资金后再买入）
- 指令 vs 实际的滑点统计

验证目标：
- 指令是否合理（股数、方向、金额）
- 真实滑点与模拟滑点的差距
- 手动执行的操作负担是否可接受（一年约 50 次）

周期：6-12 个月

退出条件：
- 真实滑点 < 0.2%
- 无一次指令错误
- 操作流程顺畅

## Phase 3: 半自动

执行器：`BrokerExecutor(auto_confirm=False)`
配置：`XIRANG_EXECUTOR=semi_auto`，`XIRANG_BROKER=ibkr`

行为：
- 系统通过券商 API 自动下单
- 但每笔订单需要你在手机上点"确认"
- 超时未确认自动取消

新增能力：
- 券商 API 连接（IBKR / 富途）
- 实时报价获取
- 订单提交与状态跟踪
- 部分成交处理
- 账户持仓同步（每天从券商拉取真实数据）

验证目标：
- API 连接稳定性
- 订单执行可靠性
- 持仓同步准确性

周期：3-6 个月

退出条件：
- API 连接成功率 > 99.9%
- 订单执行成功率 > 99%
- 持仓同步偏差 < 0.1%

## Phase 4: 全自动

执行器：`BrokerExecutor(auto_confirm=True)`
配置：`XIRANG_EXECUTOR=auto`，`XIRANG_BROKER=ibkr`

行为：
- 完全自动执行，无需人工干预
- 只在风控触发时推送通知
- 每天自动同步券商持仓

安全机制：
- 单笔订单金额上限（防止 bug 导致全仓操作）
- 日交易次数上限（防止死循环）
- 异常价格检测（防止闪崩误操作）
- 所有订单记录可审计

## 切换方式

只需要改一个环境变量：

```bash
# Phase 1 → Phase 2
echo "XIRANG_EXECUTOR=manual" >> .env

# Phase 2 → Phase 3
echo "XIRANG_EXECUTOR=semi_auto" >> .env
echo "XIRANG_BROKER=ibkr" >> .env

# Phase 3 → Phase 4
# 把 semi_auto 改成 auto
sed -i 's/semi_auto/auto/' .env
```

核心引擎代码零修改。
