# 上线前审计清单（Pre-flight Checklist）

## 一、量化逻辑审计

### ✅ 1. 未来函数排查（Look-ahead Bias）

状态：已确认安全。

- daily_runner 在美东收盘后（17:00）运行，使用当天收盘价计算风控和生成指令
- 当前系统已支持最小真实下单骨架，但真实执行前仍有券商同步闸门、白名单矩阵和执行后对账
- 默认不应开启真实下单开关，因此不存在“用收盘价直接成交”的设计目标
- 回测中的 0.1% 摩擦成本已覆盖执行误差

### ✅ 2. 数据源断流与脏数据容错

状态：已实现 Fail-safe。

代码位置：`engine/data_validator.py`

校验规则：
- 缺少资产数据 → 中止
- NaN / Inf → 中止
- 价格 ≤ 0 → 中止
- 单日涨跌幅 > ±25% → 中止（美股熔断极限）
- 数据拉取超时/失败 → 中止

中止时行为：
- 不执行任何调仓
- 记录失败到 daily_runs 表
- 推送错误通知（🚨 数据异常）

### ✅ 3. 除权除息与拆股

状态：已确认安全。

- 回测使用 Adjusted Close（复权价），正确反映分红再投资
- 实盘中系统基于账户总净值和持仓市值计算偏离度，不依赖单纯价格连乘
- 单日 ±25% 的熔断检测会捕获拆股导致的异常价格变动，中止运行等待人工确认

## 二、工程与系统审计

### ✅ 4. 幂等性保护

状态：已实现。

代码位置：`db/database.py` → `has_run_today()` / `record_run()`

机制：
- 每次运行前检查 daily_runs 表：今天是否已成功运行
- 如果已运行 → 跳过，返回 SKIPPED
- 运行成功 → 写入 SUCCESS 记录
- 运行失败 → 写入 FAILED 记录（不阻止次日重试）
- API 提供 force=true 参数可强制重跑

### ✅ 5. 凭证与隐私隔离

状态：已实现。

- 所有敏感信息通过环境变量配置（TELEGRAM_BOT_TOKEN 等）
- .env 文件已加入 .gitignore
- 代码中零硬编码凭证
- 数据库文件（*.db）已加入 .gitignore

### ✅ 6. 数据库备份

状态：已实现。

代码位置：`runner/daily_runner.py` → `_backup_db()`

机制：
- 每次运行成功后自动备份 xirang.db 到 db/backups/
- 文件名格式：xirang_YYYYMMDD.db
- 自动清理 30 天前的旧备份
- db/backups/ 已加入 .gitignore（备份不入仓库）

建议：生产环境额外配置异地备份（通过 Telegram Bot 发送 .db 文件）

## 三、合规与业务边界审计

### ✅ 7. 绝对单人模式

状态：已确认。

- 无用户表、无注册、无权限系统
- 数据库中只有一个 portfolio_id = 'default'
- API 无认证（仅本地访问）
- 从物理上断绝"代客理财"的可能性

### ✅ 8. 不接管交易

状态：已从“纯人工”升级为“最小真实交易能力 + 强边界控制”。

- 系统已具备 `IBKR / Futu` 的最小真实交易协议
- 真实交易默认关闭，必须显式开启环境变量
- 执行前必须通过券商同步、对账覆盖交易日、执行白名单矩阵
- 执行后必须通过对账闭环，否则进入人工复核或失败关闭
- 出金依旧不属于自动执行范围

## 审计结论

当前边界下，系统已超过纯 Paper Trading，但仍不应被视为“开放式实盘自动交易系统”。
更准确的结论是：可以停留在“小额、白名单、强审计、强 fail-closed”的观察运行阶段。
