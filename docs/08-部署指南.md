# 部署指南

## 设计理念

息壤的运行模式是"cron 驱动 + 跑完即退"：

- 不需要任何常驻进程
- 不需要 Docker / Kubernetes / 消息队列
- 一个 cron 任务 + 一个 Python 脚本 = 全自动运行
- 服务器重启后无需任何恢复操作，cron 会在下一个时间点自动执行

## 服务器要求

- 任意 Linux 服务器（最小规格即可，1核1G足够）
- Python 3.12+
- 能访问外网（拉取 Yahoo Finance 数据）

## 部署步骤

### 1. 上传代码

```bash
# 在服务器上
git clone <你的仓库地址> /opt/xirang
cd /opt/xirang
pip install -r requirements.txt
```

### 2. 配置通知（可选）

```bash
# 创建 .env 文件
cat > /opt/xirang/.env << 'EOF'
TELEGRAM_BOT_TOKEN=你的token
TELEGRAM_CHAT_ID=你的chat_id
EOF
```

### 3. 初始化数据库

```bash
cd /opt/xirang
python -m runner.daily_runner --force
```

首次运行会自动创建数据库并执行第一次更新。

### 4. 配置 cron 定时任务

```bash
crontab -e
```

添加以下内容：

```cron
# 息壤：每个工作日美东 17:00（UTC 22:00）运行
# 周末不运行（美股休市）
0 22 * * 1-5 cd /opt/xirang && /usr/bin/python3 -m runner.daily_runner >> /opt/xirang/logs/cron.log 2>&1
```

如果服务器在中国（UTC+8），美东 17:00 = 北京时间次日 05:00（夏令时）或 06:00（冬令时）：

```cron
# 北京时间每个工作日 06:00 运行（覆盖冬夏令时）
0 6 * * 2-6 cd /opt/xirang && /usr/bin/python3 -m runner.daily_runner >> /opt/xirang/logs/cron.log 2>&1
```

注意：周二到周六运行，因为北京时间周二 06:00 = 美东周一 17:00。

### 5. 验证

```bash
# 手动跑一次
cd /opt/xirang
python -m runner.daily_runner

# 查看报告
python -m runner.report

# 查看最近 30 天
python -m runner.report --days 30
```

## 日常命令

```bash
# 查看汇总报告（"过几个月回来看结果"）
python -m runner.report

# 查看最近 N 天
python -m runner.report --days 30
python -m runner.report --days 90

# 强制重跑今天
python -m runner.daily_runner --force

# 查看运行日志
tail -f logs/xirang.log

# 查看 cron 日志
tail -f logs/cron.log
```

## 可选：启动 API 服务

如果你偶尔想通过浏览器或手机查看状态：

```bash
# 后台启动 API（可选，不影响核心运行）
nohup python -m uvicorn main:app --host 0.0.0.0 --port 8000 &

# 访问
curl http://你的服务器IP:8000/api/portfolio
curl http://你的服务器IP:8000/api/health
```

API 服务是可选的。即使不启动，cron 任务也会正常运行。

## 自愈机制

- 幂等性：同一天重复运行会自动跳过
- 重试：数据拉取失败自动重试 3 次，每次间隔 60 秒
- 失败记录：失败会记录到数据库，不影响次日运行
- 数据库备份：每次成功运行后自动备份到 db/backups/，保留最近 30 天
- 日志：所有运行记录写入 logs/xirang.log

## 目录结构（服务器上）

```
/opt/xirang/
├── engine/          # 核心引擎（不要动）
├── runner/          # 运行器和报告
├── db/
│   ├── xirang.db    # 主数据库（所有状态和历史）
│   └── backups/     # 自动备份
├── logs/
│   ├── xirang.log   # 运行日志
│   └── cron.log     # cron 输出
├── .env             # 通知配置（不入仓库）
└── requirements.txt
```
