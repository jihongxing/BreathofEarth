# 系统设计

## 三层资产结构

```
[ Core Layer ]      抗通胀层（不可破坏）   占比 70-90%
[ Stability Layer ] 流动性层（缓冲器）     占比 10-20%
[ Alpha Layer ]     增长层（受限）         占比 ≤10%
```

### 抗通胀层（Core）

- 永久组合思想：股票ETF + 长期国债 + 黄金 + 现金类
- 硬规则：无杠杆、无择时、无人工干预
- 收益处理：保留复利

### 流动性层（Stability）

- 货币基金 / 短期债券
- 功能：接收入金缓冲、提供出金资金缓冲、吸收波动
- 提现预览时优先从此层测算扣减
- 该层属于家族主资产的一部分，不得与 Alpha/Lab 共用账本

### 增长层（Alpha）

- 最大资金占比 ≤ 10%
- 最大回撤 ≤ 10%，超过即熔断清仓，冷却30天
- 必须与 Core/Stability 隔离运行
- 建议独立数据库、独立 API、独立净值曲线、独立账户
- 不得共享主仓默认 portfolio_id，不得直接写入主仓状态
- 盈利是否转回主仓，必须由人工复核和治理批准

## 状态机设计

### 全局状态

```
INIT → IDLE（正常运行）

IDLE → INFLOW（入金）→ IDLE
IDLE → REBALANCING（调仓）→ IDLE
IDLE → WITHDRAW_REQUEST（出金请求）→ WAIT_MANUAL_EXECUTION

任意状态 → RISK_ALERT → PROTECTION → IDLE
```

### Core 层子状态

```
CORE_IDLE → CORE_DRIFTING（偏离>阈值）→ CORE_REBALANCING → CORE_IDLE
任意 → CORE_LOCKED（极端风险）→ CORE_IDLE
```

### Alpha 层子状态

```
ALPHA_IDLE → ALPHA_ACTIVE → ALPHA_DRAWDOWN → ALPHA_STOPPED（冷却）→ ALPHA_IDLE
```

## 资金流

### 入金

```
用户充值 → 自动进入 Stability（缓冲）→ 满足条件时补充 Core
```

### 收益流动

```
Core 收益 → 保留（复利）
Alpha 收益 → 默认留在 Alpha 独立账本，是否回流必须人工治理批准
```

所有正式主仓收益只认 Core/Stability 的真实账本闭环。

### 出金

```
用户发起请求 → 冷静期 / 审批 → 人工卖出与人工划转 → 人工回填执行结果
```

原则：

- 系统可以自动化入金分配
- 系统不可以自动化出金执行
- 出金属于治理行为，不属于自动交易行为

### Core 调仓执行边界

```
行情与账户同步 → 风控检测 → 券商同步硬闸门 → 白名单矩阵 → 小额真实执行 → 执行后对账
```

原则：

- 默认少打扰，但不是无条件放行
- 只有数据过期、成交偏差过大、券商回执缺失等白名单异常才升级人工介入
- 任何未通过执行前闸门或执行后对账的结果，均不得直接写成“已稳定完成”

## 再平衡规则

### 触发条件

- 阈值触发：任一资产偏离度 > 20%
- 时间触发：最小间隔 90 天，强制执行 365 天
- 现金流触发：大额入金/出金

### 执行顺序

1. 用新资金补低配资产
2. 再卖高配资产
3. 最后整体校准

原则：尽量减少卖出（降低税/摩擦）。

---

## 风控引擎

风控优先级 > 一切交易。

### ① 相关性崩溃检测

- 触发：股票-债券相关性 > 0.5（持续30天）
- 行为：进入 PROTECTION，提高现金比例 +10%，暂停再平衡

### ② 极端回撤保护

- 触发：Core 层回撤 > -15%
- 行为：停止卖出安全资产，暂停 Alpha

### ③ 流动性风险

- 触发：Stability 层 < 5%
- 行为：强制补充现金，限制出金

### ④ Alpha 熔断

- 触发：Alpha 回撤 > -10%
- 行为：清仓，冷却30天

## 周期调度

| 周期 | 任务 |
|------|------|
| 每日 | 更新价格、风控检测 |
| 每月 | 收益评估、轻提醒 |
| 每季 | 检查再平衡条件 |
| 每年 | 强制再平衡、参数校准 |
