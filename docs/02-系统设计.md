# 系统设计

## 三层资产结构

```
[ Core Layer ]      抗通胀层（不可破坏）   占比 70-90%
[ Stability Layer ] 流动性层（缓冲器）     占比 10-20%
[ Alpha Layer ]     增长层（受限）         占比 ≤10%
```

### 抗通胀层（Core）

- 永久组合思想：股票ETF + 长期国债 + 黄金 + 现金类
- 硬规则：无杠杆、无择时、无人工干预
- 收益处理：保留复利

### 流动性层（Stability）

- 货币基金 / 短期债券
- 功能：接收入金缓冲、提供出金资金、吸收波动
- 提现时优先从此层扣除

### 增长层（Alpha）

- 最大资金占比 ≤ 10%
- 最大回撤 ≤ 10%，超过即熔断清仓，冷却30天
- 盈利自动转入：70% → Core，30% → Stability

## 状态机设计

### 全局状态

```
INIT → IDLE（正常运行）

IDLE → INFLOW（入金）→ IDLE
IDLE → REBALANCING（调仓）→ IDLE
IDLE → WITHDRAWAL（出金）→ IDLE

任意状态 → RISK_ALERT → PROTECTION → IDLE
```

### Core 层子状态

```
CORE_IDLE → CORE_DRIFTING（偏离>阈值）→ CORE_REBALANCING → CORE_IDLE
任意 → CORE_LOCKED（极端风险）→ CORE_IDLE
```

### Alpha 层子状态

```
ALPHA_IDLE → ALPHA_ACTIVE → ALPHA_DRAWDOWN → ALPHA_STOPPED（冷却）→ ALPHA_IDLE
```

## 资金流

### 入金

```
用户充值 → 进入 Stability（缓冲）→ 按目标权重分配到三层
```

### 收益流动

```
Core 收益 → 保留（复利）
Alpha 收益 → 70% 转入 Core + 30% 转入 Stability
```

所有盈利最终"加固核心"。

### 出金

```
优先从 Stability 扣除 → 不足再卖 Core 超配资产
```

## 再平衡规则

### 触发条件

- 阈值触发：任一资产偏离度 > 20%
- 时间触发：最小间隔 90 天，强制执行 365 天
- 现金流触发：大额入金/出金

### 执行顺序

1. 用新资金补低配资产
2. 再卖高配资产
3. 最后整体校准

原则：尽量减少卖出（降低税/摩擦）。

---

## 风控引擎

风控优先级 > 一切交易。

### ① 相关性崩溃检测

- 触发：股票-债券相关性 > 0.5（持续30天）
- 行为：进入 PROTECTION，提高现金比例 +10%，暂停再平衡

### ② 极端回撤保护

- 触发：Core 层回撤 > -15%
- 行为：停止卖出安全资产，暂停 Alpha

### ③ 流动性风险

- 触发：Stability 层 < 5%
- 行为：强制补充现金，限制出金

### ④ Alpha 熔断

- 触发：Alpha 回撤 > -10%
- 行为：清仓，冷却30天

| 周期 | 任务 |
