# Phase 2 风控引擎调参全记录

## 起点：Phase 1 的问题

Phase 1（纯被动永久组合，无风控）的结果：

| 指标 | 值 |
|------|-----|
| CAGR | 7.42% |
| MDD | -17.20% |
| 夏普比率 | 0.75 |
| 调仓次数 | 31 |

三项成功标准中，MDD 未通过（-17.20% > -15% 目标），超标 2.2 个百分点。

---

## 第一轮：基础状态机（你提供的初始参数）

引入双模式状态机（IDLE / PROTECTION）和两个风控触发器。

参数：
- 回撤阈值：-12%
- 相关性阈值：SPY-TLT 30日滚动相关性 > 0.5
- 保护权重：SPY 10% / TLT 20% / GLD 20% / SHV 50%
- 冷却期：20天

结果：

| 指标 | Phase 1 | 第一轮 | 变化 |
|------|---------|--------|------|
| CAGR | 7.42% | 7.27% | -0.15% |
| MDD | -17.20% | -15.68% | +1.52% |
| 夏普 | 0.75 | 0.76 | +0.01 |
| 调仓 | 31 | 60 | +29 |
| 保护触发 | - | 17次 | - |
| 保护天数 | - | 388天(7.3%) | - |

判断：方向正确，MDD 削掉了 1.52%，但还差 0.68%。保护模式触发 17 次偏多，怀疑相关性检测存在误触发。

---

## 第二轮：全面收紧参数（过度激进）

思路：既然差 0.68%，那就全面收紧——更早介入、更多现金、更长冷却。

参数变更：
- 回撤阈值：-12% → **-10%**（更早介入）
- 保护权重：SHV 50% → **60%**（SPY 5% / TLT 15% / GLD 20% / SHV 60%）
- 冷却期：20天 → **30天**

结果：

| 指标 | 第一轮 | 第二轮 | 变化 |
|------|--------|--------|------|
| CAGR | 7.27% | **6.32%** | -0.95% |
| MDD | -15.68% | **-15.23%** | +0.45% |
| 夏普 | 0.76 | **0.67** | -0.09 |
| 保护天数 | 388天(7.3%) | **825天(15.6%)** | +437天 |

判断：**过度激进**。MDD 只多压了 0.45%，但 CAGR 暴跌近 1%，保护天数翻倍到 15.6%。系统"太胆小"，错过了太多反弹。这条路走不通。

教训：不能同时收紧所有参数，会导致系统过度防御，严重损伤长期收益。

---

## 第三轮：寻找中间点

思路：在第一轮和第二轮之间找平衡。

参数变更：
- 回撤阈值：-10% → **-11%**
- 保护权重：SHV 60% → **55%**（SPY 8% / TLT 17% / GLD 20% / SHV 55%）
- 冷却期：30天 → **25天**

结果：

| 指标 | 第二轮 | 第三轮 | 变化 |
|------|--------|--------|------|
| CAGR | 6.32% | **7.30%** | +0.98% |
| MDD | -15.23% | **-15.59%** | -0.36% |
| 夏普 | 0.67 | **0.77** | +0.10 |
| 保护天数 | 825天 | **477天(9.0%)** | -348天 |

判断：CAGR 和夏普大幅回升，但 MDD 反而比第一轮还差。说明问题不在参数的"力度"，而在相关性检测的"精度"——它在非危机时期误触发了。

---

## 第四轮：精准化相关性检测（关键转折点）

思路：之前的相关性检测只看"SPY-TLT 相关性 > 0.5"，但股债短期正相关在正常市场也会出现。真正的"股债双杀"是：相关性高 **且** 两者都在跌。

代码变更：
```python
# 之前：只看相关性
is_risk_detected = (current_dd <= threshold) or (corr > 0.5)

# 之后：相关性 + 双杀确认
corr_breakdown = (corr > 0.5) and (spy_30d_return < 0) and (tlt_30d_return < 0)
is_risk_detected = (current_dd <= threshold) or corr_breakdown
```

参数回到第一轮的基础值：
- 回撤阈值：**-12%**
- 保护权重：**SPY 10% / TLT 20% / GLD 20% / SHV 50%**
- 冷却期：**20天**

结果：

| 指标 | 第一轮(旧检测) | 第四轮(精准检测) | 变化 |
|------|----------------|-----------------|------|
| CAGR | 7.27% | **7.47%** | +0.20% |
| MDD | -15.68% | **-15.67%** | +0.01% |
| 夏普 | 0.76 | **0.78** | +0.02 |
| 保护触发 | 17次 | **10次** | -7次 |
| 保护天数 | 388天 | **247天(4.7%)** | -141天 |

判断：**精准化相关性检测效果显著**。误触发从 17 次降到 10 次，保护天数减少 141 天，CAGR 反而比 Phase 1 还高了（7.47% vs 7.42%），夏普也是全程最高的 0.78。但 MDD 仍然是 -15.67%，没压进线。

结论：相关性检测的精度问题解决了，但剩下的回撤来自一次快速暴跌（可能是 2020 年 3 月），在触及 -12% 预警之前就已经冲过了 -15%。

---

## 第五轮：收紧回撤阈值到 -10%（失败）

思路：既然 -12% 来不及反应，那提前到 -10% 介入。

参数变更：
- 回撤阈值：-12% → **-10%**
- 其他不变

结果：

| 指标 | 第四轮 | 第五轮 | 变化 |
|------|--------|--------|------|
| CAGR | 7.47% | **6.86%** | -0.61% |
| MDD | -15.67% | **-15.96%** | -0.29% |
| 保护触发 | 10次 | **15次** | +5次 |

判断：**完全失败**。MDD 反而更差了，因为 -10% 阈值导致更多误触发（15次 vs 10次），系统频繁进出保护模式，错过反弹后再次暴露在下跌中。

教训：单纯提前介入不是解决方案，过早介入 = 更多误触发 = 更差的结果。

---

## 第六轮：引入双层防线（最终方案）

思路：既然单一阈值无法同时解决"日常风控"和"极端暴跌"，那就分两层：

1. **常规保护**（-12%）：应对一般性回撤，温和减仓
2. **紧急避险**（-14%）：应对极端暴跌，激进减仓到几乎全现金

这是一个"断路器"设计——第一层是保险丝，第二层是总闸。

代码变更：
```python
# 新增硬止损阈值和紧急避险权重
HARD_STOP_DD = -0.14
WEIGHTS_EMERGENCY = [SPY 5%, TLT 10%, GLD 15%, SHV 70%]

# 状态机逻辑：
# IDLE 状态下：
#   回撤 <= -14% → 紧急避险（双倍冷却期）
#   回撤 <= -12% 或 相关性崩溃 → 常规保护
#
# PROTECTION 状态下：
#   回撤继续恶化到 -14% → 升级到紧急避险权重
```

参数：
- 回撤阈值：-12%（常规保护）
- 硬止损：-14%（紧急避险）
- 常规保护权重：SPY 10% / TLT 20% / GLD 20% / SHV 50%
- 紧急避险权重：SPY 5% / TLT 10% / GLD 15% / SHV 70%
- 冷却期：常规 20天，紧急 40天

第一次尝试（SHV 70%）：

| 指标 | 值 |
|------|-----|
| CAGR | 7.33% |
| MDD | **-15.02%** |
| 夏普 | 0.77 |

差 0.02%！把紧急避险的 SHV 从 70% 提到 **75%**：

最终结果：

| 指标 | Phase 1 | 最终 Phase 2 | 变化 |
|------|---------|-------------|------|
| CAGR | 7.42% | **7.34%** | -0.08% |
| MDD | -17.20% | **-14.81%** | +2.39% |
| 夏普 | 0.75 | **0.77** | +0.02 |
| 调仓 | 31 | **51** | +20 |
| 保护触发 | 0 | **11次** | - |
| 保护天数 | 0 | **278天(5.3%)** | - |

**三项成功标准全部通过。**

## 调参过程中的关键教训

1. **不要同时收紧所有参数**。第二轮的失败证明，全面激进 = 过度防御 = 损伤收益。

2. **精度比力度重要**。第四轮的相关性精准化是整个过程的转折点——不是检测得更多，而是检测得更准。

3. **单一阈值无法应对所有场景**。日常波动和极端暴跌需要不同的响应力度，双层防线（保险丝 + 总闸）是正确的架构。

4. **过早介入比晚介入更危险**。第五轮证明 -10% 阈值反而更差，因为误触发导致系统频繁进出，错过反弹。

5. **最终的最优解不是靠暴力调参找到的**，而是靠改变系统架构（从单层防线到双层防线）实现的。
