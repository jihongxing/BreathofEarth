# 二、三层结构设计（增强型永久组合）

### 1. 抗通胀层（核心层）

- 目标：最大程度降低波动与回撤，保持购买力
- 配置：宽基指数 ETF、长期国债、黄金 ETF、货币基金（25% x4）
- 调整原则：每周期调回目标权重（25%），风险触发可临时增加现金或黄金比例

### 2. 增强层（Alpha策略）

- 目标：在保证核心稳定性的前提下提升收益
- 设计思路：
    - 可使用市场中性策略、低波动动量策略、CTA/趋势跟踪
    - 资金占比可动态调整（例如总资金的10–20%）
    - 风控：超过回撤阈值自动平仓或回归现金层

### 3. 流动性层（现金管理）

- 目标：保证随时可出金，承担系统缓冲作用
- 资金比例：5–10%
- 工具：货币基金或银行高流动性账户
- 与抗通胀层交互：必要时提供临时资金平衡组合

# 三、状态机设计

### Portfolio 状态

IDLE → REBALANCING → PROTECTION → CASHFLOW

### 状态触发逻辑

- **REBALANCING**：周期性触发（每周或每月），偏离度 > threshold
- **PROTECTION**：回撤或风险指标触发，自动减少波动资产
- **CASHFLOW**：成员入金或出金触发，重新计算各层权重

### 每层子状态

- **抗通胀层**：NORMAL / CASHBACK / GOLDHEDGE
- **增强层**：ACTIVE / PAUSED（策略暂停）
- **流动性层**：NORMAL / FUNDING（增加现金应急）

---

# 四、资金流设计（组织/联盟自托管版）

成员入金 → 组织托管账户 → Portfolio Engine 分配到三层 → 调仓执行 → 更新成员权益

1. **入金**：成员资金进入独立账户，系统计算每人份额
2. **分层管理**：根据目标权重分配资金
3. **调仓周期**：每周/每月一次，系统自动调仓
4. **风控触发**：如果抗通胀层回撤超过阈值，自动调整权重，保护核心资产
5. **出金**：从流动性层优先支出，必要时回售抗通胀层资产

---

# 五、周期设计建议

- **基础周期**：每周一次调仓
- **风控周期**：每日监测风险指标，如回撤、波动、相关性异常
- **出金周期**：随时可发起，优先从现金层提取
- **增强层策略周期**：可更短（每日或每周），快速捕捉alpha

> 原则：周期越短 → 稳定性越高，但交易频率和成本增加。

---

# 六、风控触发设计

1. **回撤阈值**：核心层最大回撤 > 15% → 自动减仓股票/增加债券或现金
2. **相关性崩溃**：股票+债券同时下跌 → 提升黄金或现金占比
3. **增强层风险**：策略亏损超过 5–10% → 自动暂停策略
4. **出金事件**：调整各层比例保持核心稳定

---

# 七、增强型特点

- **动态分层权重**：核心层权重始终优先保障稳定
- **增强策略资金占比可调**：随系统表现动态增加或减少
- **风控优先执行**：回撤/相关性异常触发保护模式
- **流动性层可缓冲异常**：保证成员可随时出金

---

# 八、技术实现重点

- **数据库**：记录每个成员资金份额、历史调仓记录、状态机日志
- **状态机引擎**：控制 Portfolio 状态、各层子状态
- **调仓计算模块**：计算每层偏离度、生成调仓指令
- **执行接口**：ETF/API接口或者模拟交易接口
- **日志 & 回放**：完整记录每一步状态和决策，便于验证稳定性
